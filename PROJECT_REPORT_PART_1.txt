1. EXECUTIVE SUMMARY
================================================================================

This is a full-stack e-commerce web application built using the MERN stack
(MongoDB, Express.js, React, Node.js). The application provides a complete
online shopping experience with separate interfaces for customers and
administrators.

Key Features:
- User authentication and authorization
- Product browsing with filtering and search
- Shopping cart management
- Order placement with Cash on Delivery (COD)
- Product review system
- Admin dashboard for product and order management
- Banner/carousel management
- Responsive design with modern UI

Target Users:
- Customers: Browse products, add to cart, place orders, write reviews
- Administrators: Manage products, orders, and banner images

4.1 OVERALL ARCHITECTURE
------------------------

The application follows a client-server architecture pattern with clear
separation of concerns:

┌─────────────────────────────────────────────────────────────────┐
│                         CLIENT (Port 5173)                      │
│  ┌─────────────┐  ┌─────────────┐  ┌──────────────┐             │

│  │ Components  │→ │   Store     │→ │  Router      │             │

│         ↓                ↓                  ↓                   │
│  ┌──────────────────────────────────────────────┐               │
│  │          Axios HTTP Client                    │              │
│  └──────────────────────────────────────────────┘               │
└─────────────────────────┬───────────────────────────────────────┘
                          │ HTTP/REST API

4.2 Flow Example (Add to Cart):
------------------------------------

1. Frontend: User clicks "Add to Cart"
2. Frontend: Dispatch addToCart() Redux action
3. Frontend: Axios POST to /api/shop/cart/add
4. Backend: CORS middleware allows request
5. Backend: JSON parser reads request body
6. Backend: Route handler calls addToCart controller
7. Backend: Controller validates userId and productId
8. Backend: Mongoose finds or creates cart for user
9. Backend: Add/update product in cart items array
10. Backend: Save cart to MongoDB
11. Backend: Populate product details from Product model
12. Backend: Return success response with cart data
13. Frontend: Redux reducer updates cartItems state
14. Frontend: Component re-renders showing updated cart
